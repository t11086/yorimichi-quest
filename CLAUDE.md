# Whim（yorimichi-quest）

**タイトルは「Whim」**（2026-07-11 改名。旧称: よりみちクエスト）。"on a whim"＝ふとした思いつきで。英語表記のまま使い、カタカナにしない。候補 Whim / Detour からユーザーが Whim を選択。ディレクトリ名・localStorage キー（yorimichi_v1）は旧名のまま。

## 企画概要

**ターゲット**: 大人（20〜40代）の気分転換。毎日が仕事と家事のループになっていて、「無意味なことをする余白」を失っている人。子どもと一緒に遊んでも成立する。

**コンセプト**:
「ゴミをひとつ拾って捨てる」「ダンボールを丸く切る」「すべすべの石をひとつ拾う」——
そんな**役に立たない小さなクエスト**が毎日提示されるデイリークエストアプリ。
習慣化アプリが「あなたを成長させます」と迫るのに対し、これは何も約束しない。
為にならないからこそ気楽に手が動き、やってみると なぜか少しだけ機嫌が良くなる。

**キャッチコピー**: 「人生に、意味のない用事を。」

**設計原則（アンチ生産性）**:
- 効能を謳わない。「リラックスできます」「集中力が上がります」等の文言をUIに一切出さない
- ストリーク（連続日数）なし。何日サボっても何も起きない、責めない
- 完了は自己申告でOK。証明を求めない
- クエストは30秒〜5分で終わり、お金がかからないものだけ

## コアループ

1. アプリを開くと**クエストカードが1枚だけ**表示される（3枠案はユーザー判断で1枚に変更、2026-07-09）
2. 気に入らなければ**リロール（引き直し）無制限**。ボタンでもカードの左右スワイプでも引ける
3. やったら「やった！」で**即完了**。ダイアログなし、絵文字バーストの軽いエフェクトだけで次のカードが出てくる（＝1日の消化数は無制限。写真添付は2026-07-10に廃止）
4. **テキスト入力があるクエストは、カード上に入力欄が出る**（「やった」を押す前に入力。入力は任意、アルバムに残る。完了時の入力ダイアログは2026-07-10に廃止）

### リロール無制限の採用について（決定事項）

無制限を採用する。理由: リロールに制限を付けると「もったいないから引けない」という圧が生まれ、ゆるさというコンセプトの核と矛盾する。また「石を拾う」は都心では無理、「ダンボールを切る」は手元にないと無理、といった実行可能性のばらつきを吸収する安全弁がリロールなので、ここをケチると詰みが発生する。

ただし「プール全体を眺めてタスクを消費し尽くす」暴発を防ぐため、以下でガードする:

- **1枚ずつ、スライドアニメーション付きで引く**（一覧性を与えない。カードを引くガチャ感が自制を促す）
- **その日リロールで流したクエストは、同日中は再登場しない**
- **デッキは日替わりシャッフル**（日付シードで決定的に生成。「見尽くした感」の起点を作らない）
- クエスト一覧画面は作らない。プールの中身は遊びながらしか見えない

### チェーンクエスト（つづきもの）（2026-07-10 追加）

複数のクエストを順にこなすと何かが完成する仕掛け。**完成して初めて意図に気づく**のが狙いなので、正体は徹底的に隠す:

- チェーンの**現在のステップだけ**が通常クエストに混ざってデッキに入る（一度に1枚）
- 見た目は普通のカード。カテゴリチップなし、チェーンであることを示すUIなし
- ステップを完了すると、次のステップがデッキの**ランダムな位置に紛れ込む**（直後には出ない）
- 買い物系ステップ（八角・シナモン等）には**「もう家にある」ボタン**が出る。押すと完了扱い
- 完成の演出は `finale` 付きステップのダイアログのみ（「なにかが完成した！」スタンプは2026-07-10に廃止）。ダイアログは**カードが飛ぶ前に表示**され、とじると完了してカードが飛ぶ（短歌のお披露目も同じ順序）
- **チェーンのカードは連続して出ない**（直前がチェーンなら次はチェーン以外を優先して引く）
- **同じチェーンの次のステップは、間に3枚はさまるまで出ない**（クールダウン。完了・リロールで1枚消化するごとに1減る。デッキが尽きたときだけ例外的にゆるむ）（2026-07-10 追加）
- リロールで流された場合は通常クエスト同様、同日中は再登場せず翌日また混ざる

**実装済みチェーン**:
1. **台湾風醤油**（5歩）: 八角を買ってくる → シナモンを買ってくる → 花椒を買ってくる → 瓶に入れて醤油で漬ける → 前につけた台湾風醤油を料理にかけてみる
2. **コースター**（4歩）: ダンボールを丸く切る → 白いポスカを買ってくる → 丸く切ったダンボールに模様を描く（完了時に「コースターができました」ダイアログ）→ コースターにして飲み物を飲む
3. **短歌**（5歩、2026-07-10 追加）: 好きな5文字を考える → 好きな7文字 → 好きな5文字 → 好きな7文字 → 好きな7文字。各ステップに入力欄。**未入力・文字数不一致はエラーになり完了できない**（ちょうどN文字必須。空白は除いて数える、2026-07-10 追加）。ボタンは「やった」ではなく**「きめた」**（入力を送る動作なので決定系のラベル、2026-07-10）。最終ステップの「やった」で一首を先にお披露目し（「短歌、完成。」）、とじると完了。アルバムにも歌として残る
4. **デュカ**（5歩、2026-07-10 追加）: クミン → コリアンダーパウダー → ごま → ぜんぶ混ぜる → パンにオリーブオイルとデュカをつけて食べる
5. **ミントティー**（2歩、2026-07-10 追加）: ティーバッグを買う → 砂糖をどっさり入れて飲む（モロッコの飲み方）
6. **五香粉**（6歩、2026-07-10 追加）: 八角 → シナモン → 花椒 → クローブ → フェンネル → 砕いて混ぜる。完成時に「五香粉、完成。」ダイアログで気楽な使い方（ゆで卵・唐揚げ・チャーハン）を提示 ※チェーン初の finale 付き
7. **ほうじ茶チャイ**（4歩、2026-07-10 追加）: ほうじ茶 → シナモン → カルダモン → 牛乳・砂糖と煮て飲む
8. **タコススパイス**（4歩、2026-07-10 追加）: クミン → パプリカパウダー → オレガノ → 混ぜてポテトにかけて食べる
9. **エルブ・ド・プロヴァンス**（4歩、2026-07-10 追加）: タイム → ローズマリー → オレガノ → 混ぜてポテトにかけて食べる

※ステップに `finale: { stamp, text? }` を付けると、そのステップの「やった」でダイアログが出て、とじると完了する（コースター3歩目・五香粉の最終歩で使用）。finale が無いステップは演出なしで完了

### スパイス申告（2026-07-10 追加）

- クエスト40「家にあるスパイスを、ぜんぶ教えて」: カード上に13種のスパイスチップ（八角・シナモン・花椒・クローブ・フェンネル・クミン・コリアンダーパウダー・ごま・カルダモン・パプリカパウダー・オレガノ・タイム・ローズマリー）が並び、持っているものをタップで選んで「きめた」。選択内容はアルバムに残る。**一度答えたらもう出ない**（S.spices に保存。null = 未申告）
- **申告が終わるまで、スパイス系チェーン6本（台湾醤油・デュカ・五香粉・ほうじ茶チャイ・タコス・プロヴァンス）はデッキに入らない**（買い物カードの洪水対策を兼ねる）。ミントティー・コースター・短歌は封印対象外
- 申告後は、持っているスパイスの「買ってくる」ステップを**自動でスキップ**してチェーンが始まる（チェーン途中の買い物も、進行時に持っていれば読み飛ばす）。ステップの `item` フィールドとスパイス名を照合。「もう家にある」ボタンは救済として残る
- クエスト文は理由を説明しない謎カードのままにする（チェーンの正体を予告しないため）
- アルバムの「戻す」で申告を取り消すと未申告に戻る: チェーン6本を再封印してデッキから回収、自動スキップぶんの進行を取り消し（実際に完了したステップはログから数え直して維持）、申告カードをデッキに再挿入（2026-07-11 修正）
- 封印中（未申告）のチェーンステップを「戻す」した場合、進行位置は巻き戻すがカードはデッキに戻さない（申告→ステップの順で全部戻しても、封印中のカードがデッキに紛れない）（2026-07-11 修正）

※「ダンボールを丸く切る」「好きな5文字を考える」は通常プールから外して各チェーンの1歩目に移動（重複防止）
※ペース調整（1日1ステップまで等）は今後の検討課題

## 継続の仕掛け（責めないコレクション）

- **おもいでアルバム**: 完了したクエストが日付ごとに並ぶ。テキスト入力（本のタイトル、好きな文字）や完成した短歌もここに残る（写真添付は2026-07-10に廃止）。各項目に「戻す」ボタンあり（デバッグ用。完了を取り消し、同日ならdoneTodayからも除去、チェーンは最新ステップなら1歩巻き戻す）。リスト最下部に「はじめから」ボタン（デバッグ用の全リセット。確認ダイアログ→localStorage削除→リロード）（2026-07-11 追加）
- **むいみ週報**: 週に1回、無意味な統計を表示。「今週あなたは石を2つ拾いました」「猫の絵: 通算3枚」。真面目なグラフは出さない
- **称号**: 同カテゴリの累積で解锁（「すべすべ鑑定士」「ダンボール加工技師」など）。通知や達成圧はかけず、アルバムの隅にひっそり増える
- 進捗はローカル保存（localStorage）

## タスク文体ガイド（★ここがプロダクトの本体）

タスクの温度感がすべて。量産時に自己啓発臭・雑用臭に退化させないためのルール:

1. **数を指定する** — 「ひとつ」「1枚」「10歩」。数が入ると急にクエストらしくなる
2. **質感・条件をひとつ足す** — 「すべすべの」「まっすぐな」「利き手じゃないほうの」。この一語が詩になる
3. **効能を書かない** — 「〜してリラックス」は禁止。やる理由を説明しない
4. **言い切りの動詞で終える** — 「〜しよう」「〜してみませんか」ではなく「拾う」「切る」「描く」
5. **微妙に為になるものを2〜3割混ぜる** — ゴミ拾い・レシート整理など。ただし絶対に効能面を主張しない

- 良い例: 「アルミホイルで、できるだけまんまるの球を作る」
- 悪い例: 「深呼吸してリラックスしよう」（効能臭）／「部屋を掃除する」（大きい・義務・雑用）

## タスクタグ

| タグ | 値 |
|---|---|
| ばしょ | いえ / そと / どこでも |
| どうぐ | なし / 紙とペン / はさみ類 / スマホ |

**UIにはタグもフィルタも一切出さない**（2026-07-09、「極力シンプルに」の方針でおうちモードと「ばしょ」表示は実装後に廃止）。タグはデータとして保持し、将来のバランス調整用。

一部のクエストは**カード上のテキスト入力欄**に書き込める（例: 32「本を1冊選ぶ」→本のタイトル、短歌チェーンの各ステップ）。「やった」を押す前に入力する方式（2026-07-10、完了ダイアログから変更）。入力は任意で、アルバムにクエスト文と一緒に残る。

**二択クエスト**（2026-07-10 追加）: 「やった」の代わりに選択肢ボタンが出て、選んだ結果に応じたメッセージが表示される仕組み。例: 21「左手の感情線は〜のびていますか？」→「のびてる／のびてない」→ 手相占いの結果「あなたは〜な人です。」が出て、アルバムにも残る。**先に結果ダイアログを表示し、とじたらカードが飛んでいく**順序（2026-07-10 修正）。手相占いは21〜25の5パターン（感情線・運命線・頭脳線・親指の組み方・小指の長さ）。**占いカードは連続して出ない**（直前が占いなら次は占い以外を優先して引く。他に札がないときだけ例外）。**セッションの1枚目にも出ない**（アプリを開いた直後の表示が占いになる場合は別のカードに差し替え。占いはデッキに残る）。

**カメラクエスト**（2026-07-10 追加）: 26「今日履いている靴の写真を撮る」は「やった」の代わりに「📷 しゃしんを撮る」ボタンが出て、その場でカメラが起動。撮影すると縮小（長辺640px, JPEG）してアルバムに写真つきで残り、そのまま完了になる（汎用の写真添付は廃止のまま。カメラはこのタイプのクエスト限定）。アルバムでは72pxのサムネイル表示で、タップすると拡大表示（ライトボックス）になる。

## デザイン方針（2026-07-11 刷新）

「韓国の감성手帳アプリ」「10〜20代のSNS」「手書き感」がテーマ。旧デザイン（クリーム地＋テラコッタ＋丸角＋絵文字、いわゆるClaudeで作った感）は全廃。

- **紙**: 白ベース（#f4f3ee）にドット方眼。カードとダイアログは白い紙＋マスキングテープ（黄。山札は青/ピンク）＋薄い1px罫線、角丸は小さめ（4〜8px）
- **文字**: 日本語は Zen Kurenaido（Google Fonts、ペン字風）で全環境統一。オフライン時はシステム手書き系（Klee → 教科書体 → 丸ゴ）に落ちる。**ロゴ「Whim」は Bradley Hand Bold をアウトライン化したインラインSVG**（2026-07-11、ユーザーが Bradley Hand の字形を好んだため画像化して全環境統一。フォント読み込み不要）。少し傾けて、下にキャッチコピーを小さく添える。つうさんの数字は Caveat（英語手書き）
- **UI文言のルール**（2026-07-11）: **「クエスト」というゲーム用語をUIに出さない**（ユーザー意向）。呼ぶときは「ようじ」（タブ「きょうのようじ」、アルバム「つうさん N ようじ」）。リロールボタンは「スキップ」（旧「べつのにする」）。コード内の識別子・コメント・このドキュメントでは従来どおりクエストと呼んでよい
- **色**: モノクロ（インク #33312c／サブ #9b978e）＋差し色はくすみブルー #5c74a8 のみ。完了ボタンは黒ベタ
- **絵文字はUIに一切使わない**: タブは線画SVG、完了エフェクトは ✦✧＋ の手描き記号、アルバムの印は ✓（CSSで付与）。CATS の emoji/color はデータとして残っているが未使用
- **手書きディテール**: 「べつのにする」は波線下線のテキストボタン、入力欄は下線だけ（手帳の罫線）、アルバムの写真はポラロイド風（白フチ＋傾き）、スタンプはくすみブルーの傾いた枠

## 技術方針

- 単一HTML + CSS + Vanilla JS（既存プロジェクトと同様、ブラウザで開くだけで動く構成）
- クエストプールはJS内の静的配列（id, 本文, カテゴリ, ばしょ, どうぐ）
- 日替わりデッキは日付文字列をシードにした決定的シャッフル
- 写真はローカル保存（縮小してlocalStorage、容量が厳しければIndexedDB）
- 外部依存なし、オフライン動作、スマホ縦画面ファースト。**例外はフォントのみ**: Google Fonts（Zen Kurenaido / Caveat）を読み込む。オフライン時・読み込み失敗時はシステムの手書き系（クレー→教科書体→丸ゴ）にフォールバックし、機能は損なわれない（2026-07-11、環境間でフォントを統一したいというユーザー意向）
